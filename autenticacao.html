API de Autenticação

Antes da invocação de uma API, obrigatoriamente, deve-se chamar a API de Autenticação para receber um token.
Para utilizar a API de Autenticação é necessario utilizar um client key (usuário) e client secret (senha) que serão fornecidos pela V.tal
No ambiente Sandbox o tempo de expiração do token é de 1 hora e no ambiente de Produção, o tempo passa a ser de 1 dia. Após expiração, uma nova autenticação deve ser realizada.

Gerando Token

Para gerar o token de acesso é necessário utilizar a API de autenticação conforme imagem:

1. Abra o postman;

2. Utilize a Collection enviada pelos nossos colaboradores;

3. Selecione a opção "Variables" dentro da Collection;

4.Insira o Client Key e o Client Secret;

5.Clique em Save (Disquete)

6.Vá para o "Gerador de Token";

7.Clique em Send, o token será automaticamente copiando e inserido em "Variables", vistas anteriormente.
